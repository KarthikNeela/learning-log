#list of databases
->> \l in PostgreSQL shell gives the list of databases available in the system.
select datname from pg_database; this also gives you the same result!
datname in PostgreSQL is a column in the system catalog table pg_database that stores the name of a specific database. It represents the string identifier (name) used to connect to that database and is often used in SQL queries to list, filter, or manage databases within a server instance.

Constraints
#Primary key
It is used to uniquely identify a row/record in a table
They should not be null
Each table can only have ONE primary key.
#NOT NULL
While creating a table, if we add a NOT NULL constraint after declaring the data type of the column it cannot take a null value in any of its rows.
#Default value
Same like not null, if add a default value constraint, it will keep all the rows with the same value unless explicitly mentioned while inserting
Example
CREATE TABLE <table_name> (
	<column_name> <column_data_type> default <default_Value>,
	...
);

#Auto increment
Each row from the beginning gets a value that is going to increase itself as rows keep getting added.
To use this feature use ‘SERIAL’ after declaring column name.
Example
CREATE TABLE <table_name> (
	<column_name> SERIAL,
	...
);
Mostly used for IDs.. 
SERIAL stretches from 1 to 2147483647, if needed more than tha
there is something called as bigserial.
